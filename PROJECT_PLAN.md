# 🚀 개인 맞춤형 자산관리 웹페이지 구축 전략

## 1. 🧠 전문가적 분석: 사용자 이탈 방지 및 UX 강화 전략
단순한 숫자의 나열은 사용자에게 '피로감'을 줍니다. 자산관리를 지속 가능한 '즐거운 경험'으로 바꾸기 위한 핵심 장치들을 제안합니다.

### A. 게이미피케이션 (Gamification) 도입
- **레벨 시스템 (Financial Tier)**:
  - 현재 자산 규모에 따라 '새싹 부자' -> '스노우볼 러너' -> '경제적 자유인' 등 등급을 부여합니다.
  - 다음 단계까지 남은 금액을 시각적 게이지(Progress Bar)로 보여주어 동기를 부여합니다.
- **업적 배지 (Achievements)**:
  - "첫 1,000만원 달성", "3개월 연속 흑자", "지출 방어 성공" 등 마일스톤 달성 시 3D 배지 획득 효과를 줍니다.

### B. 시각적 피드백 & 감성 터치
- **인터랙티브 파티클 효과**:
  - 목표 달성 시 화면 전체에 금화나 폭죽이 터지는 파티클 애니메이션(Confetti)을 적용합니다.
- **미래 시뮬레이션 (Time Machine)**:
  - "이대로 1년만 더 모으면?" 버튼을 눌렀을 때, 변화할 미래 자산을 몽환적인 그래프 애니메이션으로 보여줍니다.
- **부정적 피드백의 최소화**:
  - 적자가 난 달은 빨간색 경고보다는 "잠시 쉬어가는 구간"처럼 부드러운 톤앤매너로 격려 메시지를 띄웁니다.

## 2. 🛠 기술 스택 설계
확장성, 보안, 그리고 생산성을 고려하여 최신 트렌드인 **T3 Stack (또는 유사 구성)**을 기반으로 설계했습니다. 현재 로컬 환경의 제약(Node.js 실행 불가 등)을 고려하여 프로토타입은 **Modern Vanilla JS + Tailwind CDN**으로 즉시 실행 가능하게 구현하지만, 실제 프로덕션 권장 스택은 다음과 같습니다.

| 구분 | 기술 | 선정 이유 |
| :--- | :--- | :--- |
| **Framework** | **Next.js 14 (App Router)** | 서버 사이드 렌더링(SSR)을 통한 초기 로딩 속도 최적화 및 SEO 유리함. 리액트 생태계의 표준. |
| **Language** | **TypeScript** | 금융 데이터의 정확성이 필수적이므로, 정적 타입 체크를 통해 수치 오류를 미연에 방지. |
| **Styling** | **Tailwind CSS** | 디자인 시스템을 빠르게 구축하고, 클래스명 고민 없이 직관적인 반응형 디자인 구현 가능. |
| **Database** | **Supabase** | Firebase의 오픈소스 대안. 관계형 데이터베이스(PostgreSQL)를 제공하여 자산 데이터의 복잡한 관계를 모델링하기 좋고, Row Level Security(RLS)로 보안성이 뛰어남. |
| **State** | **Zustand** | Redux보다 가볍고 직관적인 상태 관리 라이브러리로, 복잡한 사용자 설정값을 효율적으로 관리. |

## 3. 📊 데이터 시각화 전략
자산의 흐름을 한눈에 파악하기 위해 직관적이고 아름다운 차트를 구성합니다.

- **메인 차트 (자산 성장 곡선)**:
  - **유형**: `Area Chart` (영역 차트) + `Gradient Fill`
  - **이유**: 단순 선 그래프보다 하단을 색상으로 채워 "쌓여가는 자산의 무게감"을 시각적으로 전달합니다.
- **현금 흐름 (수입 vs 지출)**:
  - **유형**: `Stacked Bar Chart` (누적 막대 차트) 또는 `Sankey Diagram`
  - **이유**: 총 수입에서 고정비, 변동비가 얼마나 차지하고 얼마가 저축으로 이어지는지 흐름을 보여줍니다.
- **목표 달성률**:
  - **유형**: `Radial Bar` (원형 게이지)
  - **이유**: 전체 원을 100%로 두고 현재 어디까지 왔는지 직관적으로 보여주며, 중앙에 % 숫자를 배치하여 가독성을 높입니다.
